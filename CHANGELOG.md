# 변경 로그

JSON 포맷터 프로젝트의 모든 주요 변경사항을 기록합니다.

## [2.0.0] - 2025-08-06

### 🎉 주요 변경사항 - 코드 품질 개선

이번 릴리스는 코드 품질과 유지보수성을 크게 향상시키는 대규모 리팩토링을 포함합니다.

### ✨ 추가된 기능

#### 타입 안전성
- 모든 Python 함수와 메서드에 타입 힌트 추가
- MyPy를 통한 정적 타입 검사 도입
- IDE에서 향상된 자동완성 및 오류 감지

#### 로깅 시스템
- 중앙화된 로깅 시스템 구현 (`core/logging.py`)
- 환경별 로그 레벨 설정 지원
- 요청 추적을 위한 미들웨어 추가
- 구조화된 로그 포맷 적용

#### 환경 변수 관리
- 체계적인 설정 관리 시스템 (`core/config.py`)
- `.env` 파일을 통한 환경별 설정 지원
- 필수 환경 변수 검증 로직 추가
- 개발/테스트/운영 환경 분리

#### 코드 품질 도구
- Black 코드 포맷터 통합
- Flake8 린터 설정 및 적용
- Bandit 보안 검사 도구 추가
- Pre-commit 훅을 통한 자동 품질 검사

### 🏗️ 아키텍처 개선

#### 모듈화된 구조
```
src/
├── core/          # 핵심 기능 (설정, 로깅, 예외)
├── models/        # 데이터 모델
├── services/      # 비즈니스 로직
├── web/           # 웹 레이어 (라우트, 미들웨어)
└── utils/         # 유틸리티 함수
```

#### 의존성 주입
- Flask 애플리케이션 팩토리 패턴 적용
- 서비스 계층에 의존성 주입 구현
- 테스트 가능한 구조로 개선

#### 객체지향 설계
- 클래스 기반 라우트 구현 (`APIRoutes`)
- 단일 책임 원칙 적용
- 인터페이스와 구현 분리

### 🔄 마이그레이션 가이드

#### 기존 코드에서 변경된 사항

1. **파일 구조 변경**
   ```
   기존: app.py, views.py, json_formatter.py
   신규: src/ 디렉토리 구조
   ```

2. **API 엔드포인트 변경**
   ```
   기존: /format, /validate
   신규: /api/format, /api/validate, /api/comments
   ```

3. **환경 변수 추가**
   ```
   새로 추가: FLASK_ENV, LOG_LEVEL, MAX_CONTENT_LENGTH
   ```

#### 호환성
- 기존 API 엔드포인트는 여전히 작동
- 기존 환경 변수 설정 유지
- 점진적 마이그레이션 가능

### ⚠️ 주의사항

#### 개발 환경
- Python 3.8 이상 필요
- 새로운 개발 의존성 설치 필요: `pip install -e ".[dev]"`
- Pre-commit 훅 설치 권장: `pre-commit install`

#### 운영 환경
- 새로운 환경 변수 설정 필요
- 로그 파일 위치 및 권한 확인
- 보안 설정 검토 권장

- 애플리케이션 시작 시간 단축
